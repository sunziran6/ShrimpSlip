# 虾滑智能（ShrimpSlip）产品需求文档

## 1. 产品概述

### 1.1 产品简介

虾滑智能是一款 AI 驱动的智能购物电商平台。平台同时提供两种购物模式：

- **AI 智能购物**：用户用自然语言描述需求，Agent 自动理解意图、拆解品类、筛选商品、生成清单方案，一键下单
- **直接选购**：传统电商体验，用户浏览分类、搜索商品、加入购物车、自主下单

两种模式共享同一套商品库、购物车、订单和支付体系。

### 1.2 核心价值

- **降低决策成本**：不知道买什么时，一句话就能得到完整购物方案（AI 模式）
- **场景化购物**：按场景（露营、搬家、新生儿等）自动组合多品类商品（AI 模式）
- **对话式交互**：方案不满意可以追问修改，像跟导购聊天一样（AI 模式）
- **高效自主选购**：明确需求时，直接搜索比价下单，不绕弯（直接选购模式）
- **双模式无缝切换**：AI 生成的方案可手动调整，浏览时也可唤起 AI 帮忙决策

### 1.3 目标用户

| 用户类型 | 特征 | 典型场景 |
|----------|------|----------|
| 场景购物者 | 有明确场景但不知道具体买什么 | "我要去海边露营，帮我配一套装备" |
| 比价用户 | 知道要买什么，但想找最优方案 | "3000 元左右哪款手机性价比最高" |
| 懒人用户 | 懒得搜索筛选，想一键搞定 | "帮我买下周的食材，两人份" |

## 3. 功能需求

### 3.1 AI 智能购物（P0 核心，默认入口）

用户进入平台首先看到 AI 对话界面，这是平台的主入口和核心体验。用户用自然语言描述购物需求，Agent 自动完成全流程。直接选购作为补充路径，在用户明确知道自己要买什么时提供传统浏览体验。

| 编号 | 功能 | 描述 |
|------|------|------|
| F-AI-01 | 自然语言输入 | 用户以自然语言描述购物需求，支持文本输入和语音输入 |
| F-AI-02 | 意图识别 | Agent 自动识别购物场景、品类范围、预算约束、时间要求 |
| F-AI-03 | 品类拆解 | Agent 将模糊需求拆解为具体商品品类清单，每品类附推荐权重 |
| F-AI-04 | 智能检索 | Agent 调用商品搜索 API，按品类+预算+评分多维筛选 |
| F-AI-05 | 方案生成 | Agent 组合商品生成购物清单，含总价、推荐理由、替换备选 |
| F-AI-06 | 流式返回 | Agent 生成内容通过 SSE 实时推送到前端，用户可看到生成过程 |
| F-AI-07 | 多轮对话 | 用户可追问、替换、调整方案，Agent 保持上下文记忆 |
| F-AI-08 | 一键下单 | 用户确认方案后，将清单商品批量加入购物车并跳转下单 |
| F-AI-09 | 对话历史 | 用户可查看历史对话和生成的方案，支持恢复继续 |

### 3.2 直接选购（P0 补充路径）

当用户明确知道自己要买什么、不想经过对话流程时，可直接浏览搜索商品。通过顶部搜索栏或分类导航进入，与 AI 对话模式无缝切换。

| 编号 | 功能 | 描述 |
|------|------|------|
| F-SHOP-01 | 首页 | Banner、分类入口、推荐商品、搜索栏（从 AI 对话可一键跳转） |
| F-SHOP-02 | 商品分类 | 多级分类树，点击分类查看商品列表 |
| F-SHOP-03 | 商品搜索 | 关键词搜索，支持拼音、同义词、模糊匹配 |
| F-SHOP-04 | 筛选排序 | 按价格、销量、评分、新品筛选排序 |
| F-SHOP-05 | 商品详情 | SPU 主图/轮播图、SKU 规格选择、价格、库存、评价列表 |
| F-SHOP-06 | 加入购物车 | SKU 选中后加入购物车，支持数量修改 |
| F-SHOP-07 | 购物车管理 | 修改数量、删除、批量选择、全选、合计金额 |
| F-SHOP-08 | 购物车结算 | 选中商品 → 确认订单页（地址、优惠券、运费）→ 提交订单 |

## 5. MVP 范围

### 第一期（核心闭环）

```
AI 购物（主）：  输入需求 → Agent 生成方案 → 确认清单 → 一键加购 → 下单 → 支付
                        ↑
              用户进入平台的默认入口
                        ↓
直接选购（辅）：  浏览分类 → 搜索商品 → 商品详情 → 加入购物车 → 下单 → 支付
                         ↑
                    两种模式共享购物车、订单、支付
```

- 直接选购（分类浏览、搜索、筛选、购物车、下单）
- AI 智能购物（意图识别 + 品类拆解 + 方案生成）
- 用户注册登录（手机号）
- 商品管理（平台自营，先不做商家入驻）
- 购物车 + 下单 + 微信支付
- Web 端 + 管理后台

### 第二期

- 商家入驻
- 语音输入
- 用户画像 + 个性化推荐
- 优惠券 + 促销
- 微信小程序

### 第三期

- 多轮对话增强（图片理解、链接识别）
- 智能比价
- 跨境商品
- 数据看板

## 6. 关键业务流程

### 6.1 直接选购流程

```
用户进入首页
  → 浏览分类 / 关键词搜索
  → 筛选排序（价格、销量、评分）
  → 查看商品详情（规格、库存、评价）
  → 加入购物车
  → 继续购物 / 去结算
  → 确认订单（地址、优惠券、运费）
  → 下单支付
```

### 6.2 AI 购物流程

```
用户输入需求
  → Agent 意图识别（场景、品类、预算、时间）
  → Agent 品类拆解（子任务列表）
  → Agent 并行商品检索（Tool Call → ES 搜索）
  → Agent 方案生成（清单 + 总价 + 理由）
  → 流式返回前端（用户可见生成过程）
  → 用户确认 / 追问 / 调整
  → 一键加入购物车
  → 下单支付
```

### 6.3 下单支付流程

```
购物车结算
  → 确认订单（地址、优惠券、运费）
  → 预扣库存（Redis）
  → 创建订单（状态: 待付款）
  → 发起支付（微信/支付宝）
  → MQ 异步等待支付结果
  → 支付成功：扣减库存（DB）、更新订单状态
  → 支付超时（15min）：释放库存、取消订单
```
